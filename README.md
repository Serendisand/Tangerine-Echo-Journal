# 柑音手账 Tangerine Echo Journal

一个轻量、静态、零构建的个人手账网站。  
文章放在 `docs/` 目录里，用 Markdown 编写；首页会自动读取 `docs/list.txt`，渲染成带标签、日期、搜索和分页的文章列表。

## ✨ 功能特点

- **Markdown 文章展示**：使用 Markdown 写日记、观后感、更新记录等内容。
- **文章日期显示**：从文件名中的 `_YYYY-MM-DD` 自动解析日期，例如 `五一快乐_2026-05-01.md`。
- **标签系统**：文章首行可以写 `tag:日记,假期`，首页会自动归类并显示标签筛选。
- **搜索与分页**：支持按标题、正文内容、标签搜索文章，并自动分页。
- **正文阅读页**：点击文章后进入单篇阅读视图，支持返回列表。
- **首图展示**：文章中的第一张图片会作为封面图展示在阅读页顶部。
- **代码高亮**：内置 Highlight.js，Markdown 代码块会自动高亮。
- **本地缓存**：文章内容会缓存到浏览器，提升重复访问速度。
- **响应式布局**：适配桌面端与移动端。
- **零构建部署**：不需要 Node.js、不需要打包，直接托管静态文件即可。

## 📦 技术栈

- **Vue 3**：页面状态、文章列表、搜索、分页和阅读视图。
- **Marked.js**：Markdown 解析。
- **Highlight.js**：代码块语法高亮。
- **纯 HTML / CSS / JavaScript**：无构建步骤，直接运行。
- **GitHub Actions**：自动整理文章文件名并生成文章列表。

## 📝 如何写一篇新文章

在 `docs/` 目录下新建一个 Markdown 文件即可。

推荐命名格式：

```text
文章标题_YYYY-MM-DD.md
```

例如：

```text
暑假快乐喵_2026-07-21.md
```

文章内容示例：

```md
tag:日记,假期

# 暑假快乐喵

虽然放暑假挺久了，但现在才想起来更新，原谅我 qwq~

> 虽然很忙，但是不知道自己在忙什么。
```

说明：

- 第一行 `tag:` 是可选的，用来给文章添加标签。
- 标签可以用英文逗号 `,` 或中文逗号 `，` 分隔。
- 如果正文里有 `# 一级标题`，网站会优先用它作为文章标题。
- 如果正文里没有一级标题，网站会从文件名生成标题。
- 日期来自文件名里的 `_YYYY-MM-DD`。

## 📅 日期规则

网站会从文章文件名中解析日期：

```text
五一快乐_2026-05-01.md
```

会显示为：

```text
2026年05月01日
```

如果文件名里没有日期，文章仍然可以加载，但日期不会显示，也可能影响排序。  
因此建议所有文章都使用：

```text
标题_年-月-日.md
```

## 🏷️ 标签规则

文章首行可以写标签：

```md
tag:动画,观后感
```

或：

```md
tag:动画，观后感
```

首页会自动显示标签，并支持点击筛选。

## 📋 list.txt 是什么

`docs/list.txt` 是文章索引文件，首页会根据它加载文章。

格式示例：

```text
CLANNAD_2026-04-26.md
五一快乐_2026-05-01.md
暑假快乐喵_2026-07-21.md
```

每行一个 Markdown 文件名。  
如果文章没有写进 `list.txt`，首页就不会显示它。

## 🤖 自动工作流

项目包含 GitHub Actions 工作流：

```text
.github/workflows/update-list.yml
```

它会在 `docs/` 目录变化时自动执行：

1. 扫描 `docs/` 下的文章文件。
2. 给没有标准日期后缀的文章补上 `_YYYY-MM-DD.md`。
3. 跳过 `list.txt`。
4. 重新生成 `docs/list.txt`。
5. 如果有变更，自动提交回仓库。

这样平时写文章时，即使临时提交了无后缀文件，工作流也会尝试整理成标准格式。

## 🚀 本地预览

因为浏览器直接打开 `index.html` 可能会遇到 `fetch` 读取本地文件限制，建议用一个本地静态服务器预览。

如果安装了 Python：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 🔧 常见问题

### 首页没有显示某篇文章？

检查：

1. 文章是否在 `docs/` 目录下。
2. 文件名是否以 `.md` 结尾。
3. `docs/list.txt` 里是否包含这篇文章。

### 日期没有显示？

检查文件名是否包含日期：

```text
标题_YYYY-MM-DD.md
```

例如：

```text
五一快乐_2026-05-01.md
```

### 标题前面出现 `./`？

通常是 `docs/list.txt` 里写了类似：

```text
./五一快乐_2026-05-01.md
```

现在代码会自动兼容这种情况，但推荐 `list.txt` 里直接写：

```text
五一快乐_2026-05-01.md
```

### 修改后页面没有变化？

可能是浏览器缓存或本地缓存导致。可以尝试：

- 强制刷新页面：`Ctrl + F5`
- 清除浏览器缓存
- 打开开发者工具后禁用缓存再刷新
